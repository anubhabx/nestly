# `@specord/nestjs`

`@specord/nestjs` injects Specord docs routes into an existing Nest app during bootstrap.

The usage shape is intentionally close to `SwaggerModule.setup()`, but the document is still generated by Specord's static source pipeline. The helper does not call `SwaggerModule.createDocument()`, import `@nestjs/swagger`, or inspect runtime Nest metadata.

## Default Routes

```ts
setupSpecordDocs(app);
```

| Route | Purpose |
| --- | --- |
| `/api` | Local API docs workspace |
| `/api/openapi.json` | OpenAPI 3.1 JSON |

## Bootstrap Example

```ts
import { NestFactory } from "@nestjs/core";
import { setupSpecordDocs } from "@specord/nestjs";
import { AppModule } from "./app.module";

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  setupSpecordDocs(app);

  await app.listen(process.env.PORT ?? 3000);
}

bootstrap();
```

## Override The Mount

```ts
setupSpecordDocs(app, {
  path: "/reference",
  jsonPath: "/reference/openapi.json",
});
```

## Point At Source Explicitly

When the app runs from compiled `dist/`, pass the source project and root explicitly:

```ts
setupSpecordDocs(app, {
  project: "tsconfig.json",
  root: "src",
});
```

You can also provide a document or factory when generation is handled elsewhere:

```ts
setupSpecordDocs(app, {
  document: () => cachedOpenApiDocument,
});
```

Document factories and generated documents are cached per mounted docs instance by default. Disable caching when you want the JSON route to rebuild on every request:

```ts
setupSpecordDocs(app, {
  cacheDocument: false,
});
```

## Try It

The injected UI includes a browser-local Try it panel. Because the docs route is mounted inside the same Nest app, same-origin requests can call local API routes directly when the generated OpenAPI paths match the app's runtime routes.

Specord does not store credentials or proxy requests in this mode. Browser CORS and application auth still apply.
